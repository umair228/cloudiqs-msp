# 🚀 Setup Instructions - Fix Compilation Errors

## Quick Setup (2 minutes)

If you're getting compilation errors about missing `aws-exports`, follow these steps:

### Step 1: Copy the Sample AWS Config

```bash
cd /home/runner/work/cloudiqs-msp/cloudiqs-msp

# Copy the sample aws-exports file
cp src/aws-exports.sample.js src/aws-exports.js
```

### Step 2: Install Dependencies

```bash
npm install
```

### Step 3: Start the App

```bash
npm start
```

✅ **The app should now compile successfully!**

---

## What Just Happened?

The file `src/aws-exports.js` is:
- Auto-generated by AWS Amplify when you run `amplify push`
- Listed in `.gitignore` (so it's not in the repository)
- Required by the app to compile

We provide `src/aws-exports.sample.js` (which IS in the repository) as a mock/sample that you can copy to `src/aws-exports.js` for local development.

---

## Why This File is Needed

The application imports `aws-exports` in multiple files:
- `src/App.js`
- `src/index.js`
- `src/components/Navigation/Nav.js`

Without this file, you'll get compilation errors:
```
ERROR: Module not found: Error: Can't resolve './aws-exports'
```

---

## For Production Deployment

When you're ready to deploy to AWS:

```bash
# Install Amplify CLI
npm install -g @aws-amplify/cli

# Configure AWS credentials
aws configure

# Initialize Amplify project
amplify init

# Deploy backend (generates real aws-exports.js)
amplify push
```

After `amplify push`, the mock `src/aws-exports.js` will be replaced with real AWS configuration.

---

## Troubleshooting

### Still getting "Module not found" errors?

Make sure you copied the file correctly:
```bash
# Check if the file exists
ls -la src/aws-exports.js

# If not, copy it again
cp src/aws-exports.sample.js src/aws-exports.js
```

### File exists but still errors?

Try cleaning and reinstalling:
```bash
rm -rf node_modules package-lock.json
npm install
npm start
```
